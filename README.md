# Система «Мониторинг городского автобуса» (IoT / Node-RED / ThingsBoard)

Учебный командный проект по разработке и моделированию интеллектуальной транспортной системы (IoT) для отслеживания параметров городского пассажирского транспорта в режиме реального времени.

## 📌 Описание проекта
В рамках проекта была спроектирована и реализована система мониторинга, имитирующая движение автобусов по реальным городским маршрутам. Система собирает телеметрические данные, обрабатывает их на лету с помощью цепочек правил и визуализирует ключевые показатели для оператора диспетчерской службы.

Полная документация и архитектурное описание проекта прикреплены в репозитории в файле `Мониторинг авто.docx`.

## 🛠 Технологический стек
* **ThingsBoard** — IoT-платформа для управления устройствами, сбора, обработки и интерактивной визуализации телеметрии.
* **Node-RED** — инструмент визуального программирования для создания потоков данных, генерации симуляционных логов и интеграции компонентов.
* **JavaScript** — написание пользовательских функций обработки данных в узлах Node-RED и цепочках правил (Rule Chains) ThingsBoard.
* **Linux / Bash** — развертывание среды и запуск сервисов.

## ⚙️ Реализованный функционал

### 1. Моделирование и генерация данных (Node-RED)
* Разработан симулятор движения транспортных средств, генерирующий поток JSON-данных с реальной частотой.
* Имитируются и передаются следующие телеметрические параметры:
  * Географические координаты (движение по маршруту).
  * Скорость движения.
  * Уровень топлива в баке.
  * Температурные показатели датчиков салона и двигателя.

### 2. Управление устройствами и логика обработки (ThingsBoard)
* Созданы профили и уникальные идентификаторы виртуальных устройств (автобусов).
* **Цепочка правил (Rule Chain):** Настроена автоматическая фильтрация и анализ входящих сообщений. При фиксации критических параметров (например, превышение пороговой температуры `maxTemp`) система автоматически генерирует аварийное событие (**Alarm**) и отправляет e-mail уведомление оператору. Если параметры в норме, данные сохраняются в базу как стандартная телеметрия.

### 3. Диспетчерский Дашборд (UI/UX)
Создан интерактивный веб-интерфейс, включающий:
* **Интерактивную карту** с отображением текущего местоположения автобусов (цвет маркера динамически меняется в зависимости от скорости движения).
* **Сводную таблицу** ключевых показателей по всему автопарку.
* **Экран детальной аналитики**, открывающийся при клике на конкретный автобус на карте, для детального изучения графиков телеметрии и логов тревог.
